# **App Name**: VCTechFlow

## Core Features:

- Customer Portal: Rajib (customer) can log in, submit service requests for faulty printers, and track the repair status.
- Technician Portal: Chintu (Technician) can log in, view new service requests, update the printer status (picked up, repaired, ready to deliver, delivered), and record any parts changed during the repair process. The LLM can act as a tool by evaluating whether it is the correct moment in the flow to make an option available, like closing a case, marking an order as fulfilled, etc.
- Admin Dashboard: Admin user can manage users (customers and technicians) and oversee the service workflow. Monitor technician workloads to assign a pickup order for the printers
- Real-time Notifications: Chintu (Technician) receives immediate notification of new service requests.
- Superbase Authentication: Secure user authentication via Superbase, allowing user, admin and technician logins as well as new customer account creation.
- Printer Status Tracking: Realtime status tracking available through superbase which will display on the customers personal service screen as updates are available

## Style Guidelines:

- Primary color: Strong blue (#2962FF) to communicate reliability and trustworthiness.
- Background color: Light blue-gray (#E8F0FE) to provide a clean, professional backdrop.
- Accent color: Dark Red (#8B0000) to draw attention to VC in Tech Flow.
- Body and headline font: 'Inter', a sans-serif font, to ensure maximum readability and legibility
- Use consistent and clear icons to represent different service stages (e.g., 'Request Received', 'In Progress', 'Completed').
- Clean, intuitive layouts optimized for both desktop and mobile screens, using a grid-based system.
- Subtle transitions and loading animations to enhance user experience, especially during status updates.
- Splash Screen: 'VC TechFlow' with 'V' & 'C' entering from the left sequentially, and 'TechFlow' from the right. 'VC' elements use a dark red color.
